Content area: benefits of flex

  • similar to traditional app layout rendering, a lot more powerful under the hood
  • Doesn't break. grow/shrink properties handle math for you, unlike floats
  • layout.attrs pushes for declaring the 'scaffold' in the markup and the 'paint job' in the css classes, which is a nice separation of concerns. You can look in the markup to understand the structure, and look to the classes to understand the skin/theme/look and feel. but you dont have to have both html & css open & be interpreting both at the same time to understand it all together
    • breakdown:
    • JS is for behavior
    • html is for structure (and semantics, screen reader stuff, etc)
    • css is for look & feel
    • html becomes the throw away, treat all "code" as code. JS & CSS are highly important, html is where you "snap" the parts together
    • this will prob continue to be the flow as there is more of a move to adopt web components, which angular directive, ember components, polymer, etc all are driving toward
